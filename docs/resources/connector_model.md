# customerconnect_connector_model (Resource)

Manages a NetFoundry Connector Model — a reusable template defining the applications a connector hosts, from which connectors of a provider inherit their configuration.

## Example Usage

```terraform
# Connector model with a single application and address mapping
resource "customerconnect_connector_model" "web_gateway" {
  provider_id = "00000000-0000-0000-0000-000000000002"
  name        = "web-gateway-model"
  description = "Standard gateway model exposing an internal web application"
  type        = "GATEWAY"

  applications = [
    {
      name     = "internal-web"
      type     = "HTTP"
      protocol = "TCP"

      addresses = [
        {
          listen_address = ["web.internal"]
          listen_port    = ["443"]
          target_address = "10.0.1.10"
          target_port    = "443"
        }
      ]
    }
  ]
}

# Connector model with multiple applications, including one left for
# connectors to override via required fields
resource "customerconnect_connector_model" "multi" {
  provider_id = "00000000-0000-0000-0000-000000000002"
  name        = "multi-app-model"
  type        = "DEVICE"

  applications = [
    {
      name     = "ssh-access"
      type     = "SSH"
      protocol = "TCP"

      addresses = [
        {
          listen_address = ["{{location.name}}-ssh"]
          listen_port    = ["22"]
          target_address = "10.1.0.2"
          target_port    = "22"
        }
      ]
    },
    {
      name     = "printer"
      type     = "custom"
      protocol = "TCP_UDP"
      # addresses omitted — connectors attaching this model must supply them via override
    }
  ]
}

# SDK-embedded connector model (no type/protocol/addresses on applications)
resource "customerconnect_connector_model" "sdk" {
  provider_id = "00000000-0000-0000-0000-000000000002"
  name        = "sdk-embedded-model"
  type        = "SDK_EMBEDDED"

  applications = [
    {
      name = "embedded-app"
    }
  ]
}
```

## Schema

### Required

- `name` (String) Name of the connector model.
- `provider_id` (String) Provider this connector model belongs to. Changing this forces a new resource.
- `type` (String) Type of the connector model: `DEVICE`, `GATEWAY`, or `SDK_EMBEDDED`. Changing this forces a new resource.

### Optional

<a id="nestedatt--applications"></a>
### Nested Schema for `applications`

Required:

- `name` (String) Default name of the application. A connector override may replace it; the stable identity is `key`, not `name`.

Optional:

- `addresses` (Attributes List) Service address mappings. Nullable — if omitted, the connector must supply addresses via overrides. (see [below for nested schema](#nestedatt--applications--addresses))
- `protocol` (String) Protocol (`TCP`, `UDP`, or `TCP_UDP`). Required unless the owning model is SDK_EMBEDDED.
- `type` (String) Type of the application. Required unless the owning model is SDK_EMBEDDED. Must be a seeded ApplicationType name (e.g. HTTP, HTTPS, SSH) or 'custom'.

Read-Only:

- `key` (String) Server-assigned identifier for this application. Connector overrides bind to this key. Reassigned whenever the connector model's applications are replaced.
- `overridable_fields` (List of String) Field names on a connector's applications that may be set for this application.
- `required_fields` (List of String) Field names whose value this model does not pin down, which the connector must supply via override before the application can be materialized.

### Nested Schema for `applications.addresses`

Optional:

- `allowed_ports` (List of String) Allowed forward ports or port ranges. Required when forward_port is true.

- `forward_port` (Boolean) Whether the host config forwards the intercepted port. Defaults to false.
- `listen_address` (List of String) Listen addresses (hosts) for the intercept config. Nullable — connector must supply via override if omitted.
- `listen_port` (List of String) Listen ports or port ranges for the intercept config. Nullable — connector must supply via override if omitted.
- `target_address` (String) Target address for the host config. Required when forward_address is false.
- `target_port` (String) Target port for the host config (required when forward_port is false). A {{provider|customer|location}} template variable or a concrete port.

Read-Only:

- `overridable_fields` (List of String) Field names on a connector's address override that may be set for this address.
- `required_fields` (List of String) Field names whose value this model does not pin down, which the connector must supply via override before the address can materialize.
